2. Inicializar Git Localmente
Abre la Terminal integrada en VS Code (Ctrl + ).
Inicializa un repositorio Git local en la carpeta del proyecto ejecutando el comando:
bash
git init
Usa el código con precaución.

 
3. Realizar el Primer Commit
Añade todos los archivos del proyecto al área de staging. Puedes hacerlo manualmente en el panel de Control de código fuente (icono de bifurcación en la barra lateral) o con el comando:
bash
git add .
Usa el código con precaución.

Confirma los cambios con un mensaje descriptivo. Puedes hacerlo en el panel de Control de código fuente escribiendo el mensaje y haciendo clic en el botón de confirmación (check), o con el comando:
bash
git commit -m "Initial commit"
Usa el código con precaución.

(Reemplaza "Initial commit" por un mensaje adecuado). 
4. Crear un Repositorio Remoto en GitHub
Ve a GitHub.com e inicia sesión.
Haz clic en el botón + en la esquina superior derecha y selecciona Nuevo repositorio (o New repository).
Introduce un nombre para el repositorio, añade una descripción (opcional), elige si será público o privado y haz clic en Crear repositorio. No marques la opción de añadir un README o .gitignore aquí, ya que tu proyecto ya existe localmente. 
5. Conectar el Repositorio Local con el Remoto
En la página del nuevo repositorio de GitHub, verás opciones para configurarlo. Busca la sección que indica cómo "conectar un repositorio local existente".
Copia la URL remota (normalmente HTTPS).
Vuelve a la Terminal de VS Code y ejecuta el siguiente comando, pegando la URL que copiaste:
bash
git remote add origin [URL_DEL_REPOSITORIO]
Usa el código con precaución.

Si tu rama principal no se llama main (por defecto suele ser master), puedes cambiarle el nombre a main con:
bash
git branch -M main
Usa el código con precaución.

 
6. Subir (Push) el Proyecto a GitHub 
Finalmente, sube tus cambios locales al repositorio remoto en GitHub con el comando:
bash
git push -u origin main
Usa el código con precaución.

Es posible que VS Code te pida que inicies sesión en GitHub a través del navegador para autenticarte la primera vez. 
¡Listo! Tu proyecto ahora está en GitHub. Para futuras actualizaciones, simplemente usa git add ., git commit -m "mensaje" y git push.
